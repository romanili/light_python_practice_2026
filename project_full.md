## Суть
Полный вариант практики: консольный индексатор папок на Python с SQLite. База данных используется для хранения индекса файлов, хэшей и результатов проверок.

## Ограничения
- Использовать стандартную библиотеку Python.
- Разрешен только `sqlite3` для работы с SQLite.
- Не использовать высокоуровневые библиотеки для обхода файловой системы, поиска дубликатов и генерации отчетов.
- Не сводить выполнение к одной готовой команде или внешней обертке.

## Этап 1. Каркас
**Задача:** принять путь к папке параметром запуска, собрать структуру проекта, описать формат данных.

SQLite создается при запуске утилиты. На этом этапе нужно заложить минимальную схему таблиц под индекс файлов.

**Критерий сдачи:**
- программа запускается;
- принимает путь к папке;
- основной код расположен внутри `src/`;
- есть точка входа `src/main.py`;
- заполнен `src/student_meta.md`;
- база SQLite создается без ошибок;
- есть черновой README или описание запуска.

## Этап 2. Сканирование
**Задача:** рекурсивно обойти папку и собрать путь, размер, дату изменения, тип файла и простой фильтр по расширению или имени.

Результаты сканирования сохраняются в SQLite как индекс файлов.

**Как это работает:** индекс хранит относительные пути файлов. При новом сканировании найденные файлы добавляются или обновляются, а записи, которых больше нет в текущей папке, удаляются или помечаются как отсутствующие. Если запустить утилиту на другой папке, индекс просто перестроится под новый результат сканирования.

**Критерий сдачи:**
- обход папок работает;
- список файлов выводится;
- данные по сканированию сохраняются в SQLite;
- данные по сканированию понятны и проверяемы;
- работает хотя бы один простой фильтр.

## Этап 3. Дубликаты
**Задача:** посчитать хэш файлов и найти повторяющиеся.

Хэши сохраняются в SQLite, чтобы повторно использовать результаты между запусками.

**Критерий сдачи:**
- для файлов считается хэш содержимого;
- хэши сохраняются в SQLite;
- в отчете выводятся группы из двух и более файлов с одинаковым хэшем;
- результат можно показать в консоли.

## Этап 4. Резервная копия
**Задача:** сравнить исходную папку с бэкапом и показать различия.

Результаты проверки сохраняются в SQLite, чтобы можно было посмотреть историю запусков.

**Критерий сдачи:**
- сравнение работает;
- видны отсутствующие, измененные и лишние файлы;
- результат проверки сохраняется в SQLite;
- есть итоговый отчет.

## Итог
Рабочий вариант: утилита принимает путь, обходит папку, фильтрует файлы, находит дубликаты, сравнивает резервную копию, хранит индекс и историю запусков в SQLite.
