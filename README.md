# Практика по Python, 2026

Репозиторий для учебной практики. Здесь есть три варианта задания:

- [`project_full.md`](project_full.md) — полный вариант.
- [`project_full_easy.md`](project_full_easy.md) — упрощенный вариант.
- [`project_light.md`](project_light.md) — базовый вариант для зачета.
- [`HOW_TO_WORK.md`](HOW_TO_WORK.md) — как работать с репозиторием, журналом и коммитами.
- [`SQLITE_GUIDE.md`](SQLITE_GUIDE.md) — рекомендации по `sqlite3`, инициализации и базовым SQL-командам.

Сначала прочитайте [`HOW_TO_WORK.md`](HOW_TO_WORK.md), затем выберите один из вариантов: [`project_full.md`](project_full.md), [`project_full_easy.md`](project_full_easy.md) или [`project_light.md`](project_light.md).

## Идея
Основной проект — консольный индексатор папок на Python.  
Базовый и упрощенный варианты выполняются без базы данных.  
SQLite обязателен только в полном варианте.

## Формат кода
- Код хранится внутри `src/`.
- Точка входа проекта — `src/main.py`.
- Логику можно выносить в отдельные модули внутри `src/`.
- В полном варианте база данных создается при запуске утилиты.

## Ограничения
- Использовать только стандартную библиотеку Python.
- В полном варианте использовать SQLite через модуль `sqlite3`.
- В упрощенном и базовом вариантах SQLite не используется.
- Не использовать высокоуровневые библиотеки для сканирования файловой системы, фильтрации и отчетов.
- Не использовать фреймворки и готовые решения, которые сводят задачу к одной команде.

## Что важно
- Код писать внутри `src/`.
- Каждый этап сдавать отдельно.
- После этапа фиксировать результат в журнале.
- Коммиты делать небольшими и осмысленными.

## Как пользоваться
1. Открыть нужный план: [`project_full.md`](project_full.md), [`project_full_easy.md`](project_full_easy.md) или [`project_light.md`](project_light.md).
2. Работать по этапам.
3. Сверяться с критериями приемки в конце каждого этапа.

## Выбор варианта
- **Полный** — индексатор с SQLite, сохранением индекса и более строгими критериями.
- **Упрощенный** — индексатор без SQLite, но с дубликатами и проверкой резервной копии.
- **Базовый** — короткий вариант без SQLite, дубликатов и резервных копий.

## SQLite
Если выбран полный вариант, сначала посмотрите [`SQLITE_GUIDE.md`](SQLITE_GUIDE.md). Там есть рекомендации по созданию базы, схеме и базовым командам.
