# SQLite Guide

## Зачем нужен SQLite
- SQLite обязателен в полном варианте практики.
- В упрощенном и базовом вариантах SQLite не используется.
- Использовать только модуль `sqlite3` из стандартной библиотеки Python.
- ORM и высокоуровневые обертки не использовать.

## Что сделать в начале
1. Создать файл базы данных, например `data/app.db`.
2. Описать схему таблиц в коде или в отдельном SQL-слое, если это удобно.
3. Сделать шаг инициализации при запуске утилиты.
4. Проверить, что база создается и открывается без ошибок.

## Базовые рекомендации
- Хранить схему в строке SQL или в отдельной функции инициализации.
- Не смешивать создание схемы, загрузку данных и бизнес-логику в одном месте.
- Использовать понятные имена таблиц и полей.
- Сразу проверять `commit()` после изменений.
- Закрывать соединение после работы.

## Минимальный набор команд
- создание базы;
- создание таблиц;
- вставка тестовой записи;
- выборка списка;
- обновление записи;
- удаление записи.

## Что должно быть в учебном проекте
- понятная структура таблиц;
- отдельный шаг инициализации;
- простые запросы `SELECT`, `INSERT`, `UPDATE`, `DELETE`;
- минимум магии и минимум абстракций.

## Как живет индекс
- Индекс хранит относительные пути файлов внутри сканируемой папки.
- При сканировании новые файлы добавляются в индекс.
- Если файл уже есть, его данные обновляются.
- Если файл исчез из папки, его запись можно удалить или пометить как отсутствующую.
- При повторном запуске индекс должен отражать текущее состояние выбранной папки.
- Если запустить утилиту на другой папке, индекс перестроится под новый набор относительных путей.
