## Суть
Упрощенный вариант практики. Основной проект — консольный индексатор папок на Python без базы данных.

## Отличие от полного варианта
- Меньше обязательных деталей в выводе и отчетах.
- SQLite не используется.
- Для дубликатов достаточно показать группы одинаковых файлов.
- Для резервной копии достаточно сравнить два списка файлов без истории проверок.
- Не требуется подробно прорабатывать формат данных и сценарии повторных запусков.

## Ограничения
- Использовать стандартную библиотеку Python.
- Не использовать высокоуровневые библиотеки, которые сразу решают сканирование, фильтрацию или отчет.
- Не заменять собственную реализацию готовым инструментом.

## Этап 1. Каркас
**Задача:** принять путь к папке параметром запуска, собрать структуру проекта.

**Критерий сдачи:**
- программа запускается;
- принимает путь к папке;
- основной код расположен внутри `src/`;
- есть точка входа `src/main.py`;
- заполнен `src/student_meta.md`.

## Этап 2. Сканирование
**Задача:** обойти папку и собрать основные метаданные файлов: путь, размер, дату изменения.

**Критерий сдачи:**
- обход папок работает;
- список файлов выводится.

## Этап 3. Дубликаты
**Задача:** посчитать хэш файлов и найти повторяющиеся.

Результаты хранятся в памяти текущего запуска: `хэш -> список путей`.

**Критерий сдачи:**
- для файлов считается хэш содержимого;
- в отчете выводятся группы из двух и более файлов с одинаковым хэшем.

## Этап 4. Резервная копия
**Задача:** сравнить исходную папку с бэкапом и показать различия.

**Критерий сдачи:**
- сравнение работает;
- видны отсутствующие, измененные и лишние файлы.

## Итог
Рабочий вариант: утилита принимает путь, обходит папку, показывает основные метаданные, группирует дубликаты, сравнивает папку с резервной копией и формирует короткий отчет.
